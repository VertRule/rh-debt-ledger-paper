# R28 Policy: DOI Usage

## What the DOI Represents

The Zenodo DOI provides:
- A permanent, citable identifier for this repository
- Archival snapshot via CERN infrastructure
- Version tracking via concept DOI

## What the DOI Does NOT Represent

- **Not peer review** — Zenodo is an archive, not a journal
- **Not arXiv posting** — arXiv endorsement is a separate track (R27)
- **Not validation** — DOI existence does not validate claims

## Citation Guidelines

**For citing a specific version:**
```
Ingle, D. (2026). rh-debt-ledger-paper (r28-zenodo-doi). Zenodo. https://doi.org/10.5281/zenodo.18318664
```

**For citing all versions (recommended):**
```
Ingle, D. (2026). rh-debt-ledger-paper. Zenodo. https://doi.org/10.5281/zenodo.18318663
```

## Version Updates

When new releases are created:
1. Zenodo automatically archives each GitHub release
2. Each release gets a new version DOI
3. The concept DOI always resolves to the latest version
4. Update `CITATION.cff` with new version DOI if citing specific version
